# 项目实战经验

## 一、项目准备

### 1.1 如何选择项目
- 选择有亮点的项目：高并发、分布式、AI 集成
- 项目规模适中：能讲清楚核心流程和技术选型
- 与目标岗位匹配：后端项目侧重架构和性能，前端项目侧重交互和体验
- 避免过于常见的项目：如纯 CRUD 的管理系统

### 1.2 项目亮点提炼
- 技术深度：用到哪些核心技术，解决了什么问题
- 性能优化：做了哪些优化，效果如何（用数据说话）
- 难点攻克：遇到了什么困难，如何解决的
- 架构设计：为什么这样设计，考虑了哪些因素

## 二、后端项目常见亮点

### 2.1 用户系统
- 登录注册：Session vs JWT、OAuth2.0 第三方登录
- 权限管理：RBAC 模型、Spring Security / Sa-Token
- 用户画像：标签系统、用户匹配算法

### 2.2 缓存设计
- 缓存策略：Cache Aside、Read/Write Through
- 热点数据缓存：Redis + 本地缓存（Caffeine）多级缓存
- 缓存一致性：延迟双删、Canal 监听 binlog
- 大 Key 问题：拆分、压缩、惰性删除

### 2.3 高并发方案
- 限流：令牌桶、滑动窗口、Sentinel
- 熔断降级：Hystrix、Sentinel
- 异步处理：消息队列（RabbitMQ、Kafka）
- 分布式锁：Redis、Zookeeper、Redisson

### 2.4 数据库优化
- 分库分表：ShardingSphere
- 读写分离：主从架构
- 慢 SQL 优化：索引优化、SQL 重写
- 数据迁移：双写方案、灰度发布

## 三、前端项目常见亮点

### 3.1 性能优化
- 首屏优化：路由懒加载、骨架屏、SSR
- 接口优化：请求合并、数据缓存、防抖节流
- 打包优化：Tree Shaking、代码分割、CDN

### 3.2 用户体验
- 响应式布局：移动端适配
- 交互优化：加载状态、错误处理、空状态
- 无障碍访问：ARIA 标签

## 四、项目面试高频问题

### 4.1 通用问题
- 请介绍一下你的项目
- 你在项目中负责哪些模块
- 项目中遇到的最大挑战是什么，如何解决的
- 如果让你重新设计，你会做哪些改进

### 4.2 技术深度问题
- 你的项目用了什么技术栈，为什么选这些
- 数据库表是怎么设计的，为什么这么设计
- 接口是怎么设计的，考虑了哪些因素
- 项目的并发量是多少，怎么支撑的
- 有没有做过性能测试和优化

### 4.3 场景题
- 如果用户量增长 10 倍，你的系统能扛住吗
- 如果某个服务挂了，系统会怎样
- 如何保证数据的一致性
- 如何处理分布式事务

## 五、简历项目描述模板

### 格式
```
项目名称 | 技术栈 | 时间
项目简介：一句话描述项目做什么
核心职责：
1. 负责 XX 模块的设计与开发，使用 XX 技术解决了 XX 问题
2. 实现了 XX 功能，性能提升了 XX%
3. 设计了 XX 方案，支撑了 XX 级别的并发
技术亮点：
- 使用 XX 实现了 XX，解决了 XX 问题
- 引入 XX 优化了 XX，XX 指标从 X 提升到 Y
```

### 注意事项
- 用数据说话：QPS、响应时间、并发量、用户量
- 突出个人贡献，而非团队成果
- 技术名词要能深入解释原理
- 准备好被追问每一个技术细节
